# DOCX Report Deliverables

Use `.docx` for retrospectives, closing reports, written explanations, copy reviews, meeting-ready documents, and before/after copy comparison.

## Default Sections
1. Cover title
2. Decision-maker summary
3. Task background and known inputs
4. Multi-role consensus
5. Main findings
6. Detailed analysis
7. Action plan
8. Missing data / appendix

## Retrospective Report Structure
- 核心结论
- 数据表现
- 关键归因
- 问题诊断
- 下阶段动作
- 待补数据
- 汇报对象/客户可读口径

## Copy Review Structure
- 原文
- 原文问题诊断
- 修改方向
- 修改后版本
- 备选标题/开头
- 平台适配建议
- 风险提示

## Formatting Guidance
- Use clear section headings.
- Keep decision-maker summary near the top.
- Use tables for action items and missing data.
- Keep collaboration notes short; final content is more important than process.
