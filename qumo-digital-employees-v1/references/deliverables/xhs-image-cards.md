# Xiaohongshu / Rednote Image Card Deliverables

Use this when the user asks for 小红书图文, Rednote cards, carousel images, covers, poster images, social cards, or visual-first beauty/fashion content.

## Default Deliverables
- `.png` or `.jpg` cards for publishing.
- A source draft containing title, subtitle, body copy, page split, visual direction, and image prompts.
- Optional `.docx` review version when the user needs before/after copy, rationale, or approval notes.
- Optional `.pdf` contact sheet when the user needs one file to review all cards.

## Default Canvas
- Xiaohongshu carousel: 1080 x 1440, 3:4.
- Square social card: 1080 x 1080.
- Wide cover/reference card: 2100 x 900 or platform-specific size when supplied by the user.

## Page Flow
1. Cover: one sharp promise or scene hook, strong visual anchor.
2. Problem/scene: make the target user recognize themselves.
3. Product or method: explain what solves the problem.
4. Proof/detail: texture, usage step, comparison, data, checklist, or quote.
5. Selection/suitability: who it is for, when to use, what to avoid.
6. Action: save, comment, enter shop, watch livestream, ask for list, or next-step CTA.

## Beauty/Fashion Visual Rules
- Prefer real product, texture, skin/finish detail, outfit/accessory matching, bathroom/vanity/bag/commute/social scenes, or clean editorial still life.
- Avoid purely decorative gradients, vague luxury mood, and unreadable text-heavy posters.
- Use platform-native phrasing: specific, diary-like, experience-based, and not over-polished.
- Keep claims safe. If evidence is missing, use usage-feel and visual-effect wording instead of guaranteed efficacy wording.

## Layout Checklist
- One main message per card.
- Cover has clear hierarchy: title, subtitle, visual anchor, small supporting cue.
- Body cards use short lines, large enough text, and enough margin for mobile viewing.
- Repeated visual system: palette, type hierarchy, grid, page number style, and CTA treatment.
- Export test: view at phone size; reject cards with cramped text, poor contrast, or broken line breaks.

## Role Ownership
- `策略总监`: decide the card set's commercial purpose and core hook.
- `内容策划`: split the copy into pages and keep the note believable.
- `创意总监`: define visual concept, cover style, and image prompts.
- `PPT/报告设计师`: turn the pages into stable layouts and export-ready assets.
- `客户经理`: check risky claims, tone, and client-facing acceptability.
